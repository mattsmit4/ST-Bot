What is ST-Bot?

ST-Bot is a conversational AI assistant built to help customers navigate StarTech.com's product catalog of 5,162 technology products. Instead of browsing filters and spec sheets, customers describe what they need in plain English and the bot finds the right products.

Built with Streamlit, powered by GPT-4o-mini for intent classification, filter extraction, response generation, and narrowing. The search engine uses a two-phase scoring system — mandatory gates eliminate wrong products instantly, then scored filters rank remaining matches 0.0–1.0 by relevance. Categories are managed from a single source of truth in core/category_config.py, with LLM prompts generated from config rather than hard-coded.

Architecture (v2.0)

The pipeline follows a 6-layer flow:

  • 1. Storefront — Streamlit chat UI, the only layer the customer sees
  • 2. Sorter — Sanitizes input, loads conversation state, prevents injection
  • 3. Intent Classifier — LLM identifies the type of question (8 intents, 2 paths)
  • 4. Filter Extractor — LLM pulls specs from the query into a structured dataclass
  • 5. Handlers — One handler per intent, each with specialized logic
  • 6. Assigned Outputs — Each handler maps to its own output; assignments can be re-wired independently

Testing

Three layers of automated testing protect against regressions:

  • Unit tests (pytest tests/test_search_engine.py tests/test_derived_fields.py) — 62 fast tests for search engine scoring, connector matching, and field normalization.
  • Integration tests (pytest tests/test_integration.py) — 22 mocked tests for the full pipeline (handlers, context, state management).
  • End-to-end test harness (python tests/test_harness.py) — Real LLM calls and real product data through the full pipeline.
